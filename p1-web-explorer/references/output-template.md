# 功能地图输出模板

生成的 Markdown 文件应包含以下章节结构。根据实际探索结果增减章节。

## 文件头部

```yaml
---
title: {项目名}功能地图
date: YYYY-MM-DD
category: 工作
tags: [{项目名}, 功能测试, 功能地图]
status: done
---
```

## 章节结构

### 一、页面清单与 URL

表格列出所有页面：序号、页面名称、URL 路径、页面类型（列表页/详情页/表单页/配置页/数据分析页）。

### 二、全局导航体系

分三层记录：
1. **顶部导航栏** — 返回首页、模块标识、工作台、通知、用户信息
2. **左侧子导航** — 当前模块内的页面菜单
3. **全局产品导航** — 一级产品列表，标注当前所属产品

### 三、各页面详细结构

每个页面一个三级标题，包含：

```
### 3.N 页面名称（页面类型）

#### 筛选条件
表格：字段 | 类型 | 选项值（下拉框列出所有选项）

#### 操作按钮
表格：按钮 | 说明

#### 数据列表
表格：列名 | 说明

#### 操作按钮状态规则
列表：不同状态下哪些按钮可用/禁用

#### 分页
总条数、每页条数
```

对于详情页、新增页等特殊页面，记录：
- 面包屑路径
- 表单字段（名称、类型、必填、说明）
- 信息展示字段

对于 Tab 页签，记录每个 Tab 的名称和内容。

### 四、页面跳转关系

用树形文本描述页面间的跳转：

```text
页面A（列表页）
  ├── [按钮] → 页面B
  ├── [查看] → 页面C（详情页）
  └── 导航切换 → 页面D
```

### 五、核心业务流程

用文本流程图描述主要业务流程：

```text
步骤1 → 步骤2 → 步骤3 → 步骤4
```

### 六、数据状态流转

用文本图描述状态变化路径：

```text
状态A → 状态B → 状态C
```

### 七、关键枚举值汇总

按类别列出所有下拉选项值，方便后续设计测试用例时引用。
