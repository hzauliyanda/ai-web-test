# p1-web-explorer — 网站功能探索器

通过 Chrome DevTools MCP 控制浏览器，系统性探索 Web 项目并生成结构化功能地图。是 AI Web 测试流水线的第一步。

## 它能做什么

给定一个 Web 系统的 URL，自动：
- 探索所有页面结构和导航体系
- 提取筛选条件、下拉选项值、表单字段
- 记录数据列表、操作按钮及其状态规则
- 发现页面跳转关系和业务流程
- 汇总枚举值和状态流转
- 输出结构化的功能地图 Markdown 文件

## 前置要求

- Chrome 浏览器已安装
- Chrome DevTools MCP 插件已配置并连接
- 如果目标系统需要登录，需要手动完成登录（不自动输入密码）

## 使用方法

### 在流水线中使用

```
/ai-web-test-pipeline
→ 自动调用 p1-web-explorer 探索网站
```

### 单独使用

```
/p1-web-explorer
```

然后按提示提供：
1. **目标 URL** — 要探索的网页地址
2. **是否需要登录** — 是/否
3. **探索深度** — 完整探索（推荐）/ 仅当前页面 / 自定义
4. **输出路径** — 功能地图保存位置，默认 `{项目名}功能地图.md`

## 输入参数

| 参数 | 说明 | 必填 |
|------|------|------|
| 目标 URL | 要探索的 Web 系统地址 | 是 |
| 是否需要登录 | 系统是否需要先登录才能访问 | 是 |
| 探索深度 | 完整探索（含子页面、下拉选项、详情页）/ 仅当前页面 | 是 |
| 输出路径 | 功能地图 Markdown 文件的保存路径 | 是 |

## 输出

生成一个 Markdown 文件（如 `{项目名}功能地图.md`），包含以下章节：

| 章节 | 内容 |
|------|------|
| 一、页面清单与 URL | 所有页面的名称、URL 路径、页面类型 |
| 二、全局导航体系 | 顶部导航、左侧菜单、面包屑 |
| 三、各页面详细结构 | 每个页面的筛选条件、操作按钮、数据列表、表单字段、Tab 页签 |
| 四、页面跳转关系 | 页面之间的链接和跳转路径 |
| 五、核心业务流程 | 端到端业务操作流程 |
| 六、数据状态流转 | 数据对象的状态变化路径 |
| 七、关键枚举值汇总 | 所有下拉选项值、状态值汇总 |

## 内部工作流程

### 1. 收集信息
通过交互式问答获取目标 URL、登录要求、探索深度、输出路径。

### 2. 准备浏览器
- 调用 `list_pages` 检查 Chrome DevTools MCP 连接状态
- 连接失败时，自动杀掉旧 Chrome 进程并以调试模式重启：
  ```bash
  /Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
    --remote-debugging-port=9222 \
    --user-data-dir=/tmp/chrome-debug \
    "{目标URL}" &
  ```

### 3. 处理登录
- **需要登录**：导航到目标 URL → 截图展示 → 提示用户在浏览器中手动登录 → 等待用户确认
- **不需要登录**：直接打开目标 URL 继续

> **注意**：不会自动输入密码，因为实际项目可能有 SSO、验证码、2FA。

### 4. 探索页面
对每个页面依次执行：

| 操作 | 说明 |
|------|------|
| `take_snapshot` | 获取页面 a11y 树结构（优先，信息更结构化） |
| `take_screenshot` | 辅助截图验证 |
| 展开下拉框 | `click` combobox → 记录 listbox 内所有选项 → 关闭 |
| 进入子页面 | 点击"查看"/"新增"/"编辑"等按钮，探索详情页和表单页 |
| 切换 Tab 页签 | 点击每个 Tab 获取不同内容 |
| `navigate_page` | 访问左侧/顶部导航中的其他页面 |

提取的信息：

| 维度 | 内容 |
|------|------|
| 页面清单 | 名称、URL 路径、页面类型 |
| 导航体系 | 全局导航、子导航、面包屑 |
| 筛选条件 | 字段名、类型、所有下拉选项值 |
| 数据列表 | 列名、操作按钮、按钮状态规则 |
| 分页信息 | 总条数、每页条数 |
| 表单字段 | 字段名、类型、是否必填、默认值 |
| 页面跳转 | 页面间的链接和跳转关系 |
| 枚举值 | 所有可能的选项值和状态值 |
| 状态流转 | 数据状态的流转路径 |

### 5. 生成功能地图
按 `references/output-template.md` 模板结构生成 Markdown 文件，写入后设置文件权限。

## 目录结构

```
p1-web-explorer/
├── SKILL.md              # Skill 定义文件
├── README.md             # 本文件
└── references/
    └── output-template.md  # 功能地图输出模板
```

## 关键原则

- **先 snapshot 后 screenshot**：snapshot 信息更结构化，优先使用
- **下拉选项必须展开获取**：点击 combobox 展开完整选项列表
- **操作按钮状态规则要记录**：哪些状态下按钮可用/禁用
- **不自动输入密码**：涉及 SSO、验证码等场景，由用户手动完成登录
